# Security Policy

## Reporting a Vulnerability

Security is paramount for Flectra as we're building trust infrastructure for autonomous systems.

**DO NOT** open a public GitHub issue for security vulnerabilities.

### How to Report

1. Email: **security@flectra.xyz**
2. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

### What to Expect

- **Acknowledgment:** Within 48 hours
- **Initial Assessment:** Within 7 days
- **Resolution Timeline:** Depends on severity

### Severity Levels

| Level | Description | Response Time |
|-------|-------------|---------------|
| Critical | Direct loss of funds, protocol takeover | 24 hours |
| High | Significant impact on protocol functionality | 72 hours |
| Medium | Limited impact, workarounds available | 7 days |
| Low | Minor issues, best practices | 14 days |

## Scope

### In Scope

- Smart contracts in `/src`
- Deployment scripts in `/script`
- Any contract deployed to Base mainnet

### Out of Scope

- Frontend applications
- Third-party dependencies (report to respective maintainers)
- Already known issues in GitHub Issues

## Bug Bounty

A formal bug bounty program is coming soon. Until then, valid reports will be acknowledged and credited.

## Safe Harbor

We will not pursue legal action against researchers who:

- Act in good faith
- Avoid privacy violations
- Avoid destruction of data
- Do not exploit the vulnerability beyond proof of concept
- Report findings promptly
